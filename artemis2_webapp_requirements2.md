# Artemis II リアルタイム可視化 Web アプリ 要件定義書

## 1. 文書情報
- 文書名: Artemis II リアルタイム可視化 Web アプリ 要件定義書
- 版数: v0.1
- 作成日: 2026-04-02
- 想定利用者: 一般ユーザー、宇宙ファン、技術検証担当者
- 文書形式: Markdown

## 2. 背景
NASA は Artemis II ミッション向けに、AROW（Artemis Real-time Orbit Website）で Orion 宇宙船のリアルタイム追跡表示を提供している。加えて、state vectors および ephemeris を利用可能とし、独自アプリや可視化での活用を想定している。  
また、JPL Horizons は API 経由で軌道・位置データを取得可能であり、Artemis II ミッションの参照にも利用できる。

本アプリは、これらの公開情報を用いて、Artemis II の位置・速度・軌道進行・イベント進捗を直感的に可視化することを目的とする。

## 3. 目的
本アプリの目的は以下の通り。
1. Artemis II の現在位置をリアルタイムまたは準リアルタイムで可視化する。
2. 地球・月・宇宙船の相対位置関係をわかりやすく表示する。
3. 打ち上げから月フライバイ、帰還までのミッション進行を時系列で把握できるようにする。
4. 将来的に他ミッションにも転用可能な構造で実装する。

## 4. スコープ

### 4.1 対象範囲
- Artemis II の飛行位置データ取得
- 現在位置の可視化
- 軌跡表示
- ミッションイベント表示
- データソース切替（AROW / JPL Horizons）
- PC / モバイルブラウザ対応

### 4.2 対象外
- NASA 非公開テレメトリの取得
- 航法計算そのものの高精度再現
- 管制用途の運用画面
- ユーザー認証が必要な個人向け機能
- 投稿機能やSNS機能

## 5. 想定ユーザー
- 一般ユーザー: 「いまどこを飛んでいるか」を見たい
- 宇宙ファン: 速度、高度、月までの距離、イベント時刻なども見たい
- 開発者 / 分析者: 元データを確認しつつ可視化の再利用をしたい

## 6. データソース要件

### 6.1 優先データソース
#### A. NASA AROW 系データ
用途:
- リアルタイム表示の第一候補
- 公式表示に近い状態を優先した可視化

期待データ:
- 現在時刻
- 宇宙船位置
- 速度
- 地球からの距離
- 月からの距離
- ミッション経過時間
- イベント通過状況
- state vectors
- ephemeris

注意:
- 公開 REST API 仕様は明示されていない可能性があるため、データ取得方法は実装時に確認が必要
- JSON 等の直接取得が難しい場合は、AROW が配布する機械可読データまたはダウンロード可能な ephemeris を中間処理して扱う

#### B. JPL Horizons API
用途:
- 安定した機械可読データ取得
- 補完用 / フォールバック用
- 過去 / 将来時点を含む軌道再計算

期待データ:
- 時刻別位置ベクトル
- 速度ベクトル
- 天体基準座標
- エフェメリス

注意:
- AROW と完全一致しない座標系・更新タイミングの差異がありうる
- UI 上では「表示元データ」を明示する

### 6.2 データ更新要件
- 表示更新間隔: 初期値 5 秒
- ユーザーが切替可能な更新間隔: 1秒 / 5秒 / 10秒 / 30秒 / 手動更新
- データ取得失敗時は前回正常値を保持
- 3回連続失敗時は「現在データ取得エラー」と表示
- データソースごとに最終更新時刻を表示

### 6.3 データ保持要件
- 少なくともセッション中は過去取得点を保持し、軌跡を描画する
- 任意で 24 時間分の履歴をブラウザローカル保存可能とする
- 将来拡張としてサーバー側時系列保存を考慮する

## 7. 機能要件

### 7.1 現在位置表示
- 宇宙船の現在位置を 2D または 3D で表示する
- 地球、月、宇宙船を同一画面上に表示する
- 宇宙船アイコンは現在位置に追従する
- ユーザーはズームイン / ズームアウトできる
- ユーザーは視点を「地球中心」「月中心」「全体表示」に切り替えできる

### 7.2 軌跡表示
- 直近の軌跡を線で表示する
- 将来予測軌道が取得可能な場合は破線で表示する
- 軌跡の表示期間を切替可能にする
  - 直近10分
  - 直近1時間
  - 全ミッション
- データソース差異により予測軌道と実測表示が異なる場合は凡例を付ける

### 7.3 ミッション進行表示
- 打ち上げ
- 地球周回離脱
- 月接近
- 月フライバイ
- 地球帰還
- 再突入
などの主要マイルストーンをタイムライン表示する

各イベントについて以下を表示する:
- イベント名
- 予定時刻
- 実績時刻
- 通過済み / 未到達 / 不明 の状態

### 7.4 テレメトリ風サマリー表示
画面上に以下の主要指標を表示する:
- Mission Elapsed Time
- 現在時刻（UTC / JST 切替）
- 地球からの距離
- 月からの距離
- 相対速度
- 現在フェーズ
- データソース
- 最終更新時刻

### 7.5 データソース切替
- ユーザーは AROW / Horizons を切替可能
- 自動モードでは AROW 優先、失敗時に Horizons にフォールバック
- どのデータソースを表示しているか明示する
- 切替時に座標系差分や更新頻度差を注意書きとして示す

### 7.6 イベントログ表示
- 新しいデータ取得時刻をログ表示する
- データソース切替をログ表示する
- エラー発生をログ表示する
- ミッションマイルストーン検知時にログ表示する

### 7.7 レスポンシブ対応
- PC
- タブレット
- スマートフォン
に対応する

スマホ表示では以下を優先する:
- 現在位置
- 距離
- 速度
- 現在フェーズ
- 簡易タイムライン

## 8. 非機能要件

### 8.1 性能
- 初回表示: 3秒以内
- 通常更新処理: 1秒以内に画面反映
- 画面操作時の体感遅延は最小限とする

### 8.2 可用性
- 外部データソースの一時障害に耐える
- 前回取得値を保持して画面崩壊を防ぐ
- フロントエンドのみでも最低限の表示が継続できる設計とする

### 8.3 保守性
- データ取得層、変換層、表示層を分離する
- ミッション依存設定はコンフィグ化する
- 他ミッション追加時に UI 本体を大きく変更しなくてよい構造とする

### 8.4 拡張性
将来的に以下へ拡張可能とする:
- Artemis III 以降
- 他の NASA / JAXA ミッション
- ISS や探査機の汎用トラッカー化
- 過去リプレイ再生

### 8.5 セキュリティ
- API キーが必要な場合はサーバー側で秘匿する
- クライアントに秘密情報を埋め込まない
- 外部データの表示時は XSS 対策を行う

## 12. UI/UX 要件
- 一目で「いまどこか」が分かることを最優先
- 数値だけでなく視覚的理解を優先
- データ欠損時にも混乱しない文言にする
- 「公式データとの差異がありうる」旨を適切に表示
- 宇宙ファン向けに情報量を持たせつつ、一般ユーザーにも難しすぎない構成とする

## 13. エラーハンドリング要件
- データ取得失敗時:
  - 画面上部に警告表示
  - 最終正常更新時刻を保持表示
- データ形式不正時:
  - 当該更新を破棄
  - エラーログ記録
- AROW 取得不可時:
  - Horizons へ自動フォールバック
- 全ソース失敗時:
  - 「現在データを取得できません」と表示
  - リトライボタン表示

## 22. データ変換要件

### 22.1 正規化ポリシー
外部ソースの差異を吸収するため、内部標準モデルを定義する。

統一対象:
- 時刻: ISO 8601 UTC
- 距離: km
- 速度: km/s
- 座標フレーム: 内部標準フレームを設定し、必要に応じて変換
- phase 名称: 内部標準辞書で統一

### 22.2 phase 標準例
- Prelaunch
- Ascent
- Earth Parking Orbit
- Translunar Injection
- Translunar Coast
- Lunar Flyby
- Return Coast
- Re-entry
- Splashdown
- Complete

### 22.3 欠損時の扱い
- 距離未提供時: position から再計算
- 速度未提供時: 連続点差分から近似計算
- event 実績時刻不明時: null
- source 障害時: status=degraded を設定

- 更新周期: 5 秒
- 表示: 2D のみ
- データソース: Horizons を先に安定接続し、その後 AROW 対応

理由:
- AROW の取得方式が不透明なため、MVP ではまず取得確実性を優先する
- 軌道可視化は 2D でも多くのユースケースを満たせる
- 実装複雑度を抑えつつ要件の中核を満たせる
